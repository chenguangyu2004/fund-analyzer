# 基金分析App

基于Python开发的基金分析工具，支持实时获取基金数据、计算盈亏、分析买卖策略。

**版本**: v6.0.0 (2026-05-02)
**特点**: 无需注册、完全免费、数据全面、智能分析、AI智能投资建议

## 使用说明

### 基本操作
1. 在页面输入6位基金代码（如：000001）
2. 输入持仓成本（元/份）
3. 输入持有份额
4. 点击"分析基金"按钮
5. 查看结果：
   - 基金基本信息
   - **双净值对比**（官方收盘净值 vs 实时估算净值）
   - 盈亏计算
   - **持仓股票实时价格和涨跌幅** ✨
   - 买卖策略建议
   - 历史净值走势

### 净值数据说明

#### 官方收盘净值（🟡）
- 基金公司每日公布的正式净值
- 更新时间：交易日收盘后（通常15:00后）
- 数据来源：东方财富网
- 用途：用于正式结算、申购赎回

#### 实时估算净值（🟢）
- 基于基金持仓股票的实时价格估算
- 更新时间：交易时间实时更新（工作日9:00-15:00）
- 数据来源：天天基金网
- 用途：盘中参考、了解实时盈亏变化
- **注意**：估算值仅供参考，不作为交易依据

### 净值对比说明
当同时获取到官方和实时数据时，页面会显示：
- 官方收盘净值（黄色背景框）
- 实时估算净值（绿色背景框）
- 实时估算与官方净值的差异百分比

### 自动刷新功能
1. 分析基金后，勾选"自动刷新数据（交易时间有效）"
2. 选择刷新间隔：30秒、1分钟、2分钟、5分钟
3. 系统会自动更新数据
4. 适合交易时间关注基金表现

### 保存基金功能
1. 输入基金信息后，点击"💾 保存基金"按钮
2. 已保存的基金显示在页面顶部
3. 点击基金卡片可快速填入信息
4. 点击"刷新"可立即获取最新数据
5. 点击"删除"可移除该基金

**注意**: 已保存的基金存储在浏览器本地，清除浏览器数据会丢失。

## 常见问题

### Q: 需要注册账号吗？
A: 不需要！所有数据来自公开渠道，无需注册、无需配置任何Key。

### Q: 提示"无法获取基金信息"怎么办？
A: 请检查：
- 基金代码是否正确（6位数字）
- 网络连接是否正常
- 该基金是否为在售的公募基金

### Q: 为什么有时候没有实时估算净值？
A: 实时估算净值仅在交易时间（工作日9:00-15:00）提供，非交易时间显示官方收盘净值。

### Q: 实时估算净值准确吗？
A: 实时估算净值基于持仓股票的实时价格计算，仅供参考：
- 可能存在一定误差（股票交易与基金结算的时差）
- 不作为申购赎回的依据
- 最终以基金公司公布的官方净值为准

### Q: 持仓股票价格显示"--"是什么意思？
A: 说明该股票的实时价格暂时无法获取，可能原因：
- 股票停牌
- API请求失败
- 数据源临时不可用
- 可点击股票代码查看详情

### Q: 港股价格支持吗？
A: 支持！应用已实现港股（5位代码）的实时价格获取，例如：
- 00700 腾讯控股
- 00941 中国移动
- 02318 中国平安

### Q: 实时估算与官方净值的差异说明了什么？
A: 差异反映了盘中市场变化：
- 正差异：市场上涨，估值高于收盘价
- 负差异：市场下跌，估值低于收盘价
- 差异大小反映当日市场波动程度

### Q: 支持哪些类型的基金？
A: 支持所有公开交易的公募基金（股票型、混合型、债券型、指数型等）

### Q: 数据更新频率？
A:
- 官方净值：每个交易日更新一次，收盘后公布
- 实时估算：交易时间内实时更新
- 自动刷新：可设置30秒-5分钟

### Q: 什么时候使用自动刷新？
A: 适合以下场景：
- 交易日需要关注盘中变化
- 短线操作需要及时数据
- 监控多个基金的实时表现

### Q: 如何修改端口？
A: 编辑 `app.py` 文件，修改最后一行的端口号：
```python
app.run(debug=True, host='127.0.0.1', port=5000)  # 修改5000为其他端口
```

### Q: 已保存的基金会丢失吗？
A: 数据保存在浏览器本地，只有清除浏览器数据或使用隐私模式时才会丢失。

### Q: 修改代码后如何应用？
A: 运行以下命令重启服务器：
```bash
# Windows
start.bat restart

# Linux/Mac
./start.sh restart
```

## 开发路线图 (Roadmap)

> 以下功能按优先级分为七大部分，逐步迭代开发。

### 一、核心数据层 🔥（立即开发）

#### 1. 历史净值曲线 + 基准对比
- **为什么没有不行**：没有走势图，AI分析再强也缺直观体感
- **实现**：折线图展示净值走势，叠加沪深300/同类平均/业绩基准
- **交互**：支持缩放、拖拽、标注最大回撤区间

#### 2. 基金对比（多选并行）
- **为什么没有不行**：用户不会只看一只基金就做决策
- **实现**：支持2-5只基金同框对比（收益、风险、费率、持仓重合度）
- **展示**：对比表格 + 收益曲线叠加图 + 持仓行业差异图

#### 3. 风险指标面板
- **已有基础**：AI分析可能包含风险判断
- **补充**：独立展示夏普比率、最大回撤、波动率、索提诺比率、Calmar比率
- **可视化**：风险收益散点图（该基金在同类中的位置）

#### 4. 持仓变动追踪
- **为什么没有不行**：现在只能看当前持仓，看不到"经理调仓思路"
- **实现**：季度持仓变化对比（增仓/减仓/新进/退出股票列表）
- **价值**：识别经理是"长期持有型"还是"高频交易型"

#### 5. 完整费率结构
- **内容**：申购费（前端/后端）、管理费、托管费、赎回费阶梯、销售服务费
- **计算**：持有不同天数的赎回成本计算器

---

### 二、组合分析层 🔥（从"看基金"到"管钱"）

#### 6. 我的组合（Portfolio）
- **功能**：将保存的多只基金按实际持仓比例构建虚拟组合
- **分析内容**：
  - 组合整体收益率/盈亏
  - 行业集中度（穿透所有持仓股票计算真实行业分布）
  - 风格暴露（大盘/小盘、价值/成长）
  - 重仓股重叠度（是否买了多只基金但底层都是茅台宁德时代）
  - 相关性矩阵（基金之间是否高度相关，分散投资是否有效）

#### 7. 组合层面的AI分析
- **扩展**：让DeepSeek不仅分析单只基金，还能分析整个组合
- **场景**："我的组合科技板块占比60%，是否过于集中？"

---

### 三、工具层 🔧（提升实用性）

#### 8. 智能筛选器
- **功能**：按条件筛选基金（非输入代码，而是发现基金）
- **条件示例**："近3年夏普>1.5 + 最大回撤<20% + 规模10-100亿 + 科技主题"
- **价值**：解决"我不知道该买哪只"的问题

#### 9. 定投模拟器
- **功能**：基于历史数据回测定投收益
- **参数**：定投金额、频率、起止日期
- **输出**：定投 vs 一次性投入对比、微笑曲线可视化

#### 10. 预警/提醒系统
- **触发条件**：
  - 单日涨跌幅超过±3%
  - 基金经理变更
  - 基金规模骤增/骤减（风格漂移风险）
  - 净值跌破/涨破设定阈值
- **实现**：浏览器通知（Notification API）+ 页面内红点提示

---

### 四、AI 能力深化 🤖

#### 11. 跨基金AI问答
- **扩展**：不限于单只基金，可以问"科技主题基金最近表现如何？""比较一下易方达蓝筹和景顺长城新兴成长"

#### 12. 持仓股票AI关联分析

#### 13. 市场匹配度评分可视化
- **已有**：AI三层分析中的"市场匹配"
- **补充**：将匹配度拆解为可量化的维度雷达图（估值匹配、趋势匹配、情绪匹配、政策匹配）

---

### 五、数据持久化与性能 ⚡（技术债预防）

#### 14. 从 localStorage 升级到 IndexedDB
- **原因**：localStorage 上限约5MB，保存多只基金+历史净值很快会满
- **方案**：使用 Dexie.js 或 idb 封装 IndexedDB，支持存储大量时间序列数据

#### 15. 数据缓存策略
- **设计**：
  - 基金基础信息：缓存7天
  - 净值数据：缓存1天（交易日内）
  - 持仓数据：缓存一个季度（季报发布前不变）
- **好处**：减少API调用，提升加载速度

#### 16. 增量更新机制
- **优化**：自动刷新时只拉取最新净值，而非全量刷新所有数据

3. 端口5000是否被占用
4. 网络连接是否正常
